# Security Policy

## Our Privacy Approach
The Sovereign Recovery Suite is designed with a **Privacy-First** architecture. 
- **Local Data:** All user data (logs, settings, and streaks) is stored strictly on the user's local device using Browser LocalStorage.
- **Zero-Cloud:** There is no back-end server or database. We do not collect, see, or sell your information.
- **Privacy by Default:** This project uses no third-party trackers, analytics, or cookies.

## Supported Versions

I am committed to maintaining the security and functionality of the current active release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Active Support   |
| < 1.0   | ❌ End of Life      |

## Reporting a Vulnerability

Security is a priority. If you discover a vulnerability or a bug that affects the privacy of user data, please report it so it can be fixed.

**How to report:**
1. **Submit a Report:** Please do not use public GitHub issues for security-related bugs. 
2. **Contact:** Email SoulJah4Luv@gmail.com or open a private discussion via the project portal.
3. **Timeline:** I aim to acknowledge all security reports within 2 or 3 business days.

## Patch Process
Once a vulnerability is confirmed:
1. A fix will be developed and tested in a private branch.
2. A patch will be pushed to the `main` branch.
3. Because the app runs offline via a Service Worker, users are encouraged to refresh their browser to ensure the latest "Survival Engine" update is applied.
